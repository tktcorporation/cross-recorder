# UI大幅改善 設計ドキュメント

## 背景

現在のUIは動作面では良好だが、見た目が地味で一般ユーザーに対する訴求力が弱い。
iOS/macOS標準アプリレベルのシンプルさを維持しつつ、アニメーションでリッチに見せることで
使っている間に飽きさせないUIを目指す。

## 決定事項

| 項目 | 決定 |
|------|------|
| テーマ | ダーク維持（slate系で洗練） |
| レイアウト | シングルビュー切替型（録音 / ライブラリ） |
| アニメーション | 視覚的にリッチ（スプリング、マイクロインタラクション） |
| コンポーネント | shadcn/ui（Radix UI + Tailwind） |
| アニメーションライブラリ | Framer Motion |

## 画面構成

### 録音画面（デフォルト）

- 中央: パルスリング付きの大きな録音ボタン（音声レベル連動）
- 中段: 録音中に右へ伸びるリアルタイム横スクロール波形
- 下部: 経過時間、音声ソーストグル、デバイス選択
- 停止後: 波形がそのまま残り再生ボタンに切替 → その場で再生確認可能
- ヘッダー右: ライブラリ画面への遷移ボタン

### ライブラリ画面

- 波形プレビュー付きカードリスト
- カードタップで展開 → フル波形 + 再生コントロール
- スワイプ or ボタンで削除、フォルダを開く等のアクション
- ヘッダー左: 録音画面への戻るボタン

## アニメーション

| 要素 | アニメーション | 実装 |
|------|--------------|------|
| 画面遷移 | 横スライド + フェード | Framer Motion AnimatePresence + spring |
| 録音ボタン | ホバーでスケールアップ、タップでスケールダウン | whileHover, whileTap |
| パルスリング | 音声レベル連動でスケール・透明度変化する同心円 | useMotionValue + animate |
| リアルタイム波形 | 右方向に伸びるバー描画 | Canvas |
| 停止→再生切替 | 波形維持、ボタンモーフィング | layoutId + spring |
| カード展開 | タップで高さ拡張、再生UI表示 | layout animation |
| カード削除 | 横スライドでフェードアウト | AnimatePresence + exit |
| トグルスイッチ | スプリングベースの切替 | shadcn Switch + spring |

## カラーパレット

```
テキスト:  slate-50 / slate-400
アクセント: red-500（録音） / blue-500（再生）
ボーダー:  slate-700/50
```

## 技術スタック変更

追加:
- framer-motion: アニメーション
- shadcn/ui: UIコンポーネント（Button, Switch, Select, Card, ScrollArea等）
- tailwindcss-animate: shadcn依存
- class-variance-authority + clsx + tailwind-merge: shadcnユーティリティ

変更なし:
- React 18, Zustand 5, Vite 6, Tailwind CSS 3
- hooks, stores, RPC層（ロジック層）
